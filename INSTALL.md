# 人脉地图能力包安装契约

正式用户入口：

`https://github.com/xyxw1234-bot/relationship-map-skill`

## 接收智能体的固定动作

当用户明确说“安装人脉地图”“更新人脉地图”或发送本仓库链接时：

1. 读取本文件和 `relationship_map_plugin/plugin.yaml`，确认名称、版本和来源；
2. 只安装本仓库内的 `relationship_map_plugin`，不得安装未声明的第三方依赖；
3. 使用 Hermes 官方插件安装能力安装或更新插件；
4. 确保插件启用，并在新的 Hermes 会话或重启后的网关中加载；
5. 初始化当前用户的关系资产库；
6. 执行一次真实自检：写入一条隔离的安装自检记录后清理，不得污染用户联系人；
7. 验证捆绑 Skill 可用；
8. 只向用户回复“人脉地图已开通”及简短用法，不展示技术细节。

## 数据不丢失规则

- 插件代码与用户数据目录必须分离。
- 更新不得删除或覆盖 `$HERMES_HOME/data/relationship-map/`。
- 数据结构升级前必须创建备份；迁移失败必须保留旧库可恢复。
- 普通更新只替换插件和 Skill，不能触碰用户联系人、互动、承诺和待办。

## 用户体验

普通互动、明确承诺和待跟进事项自动沉淀。每次成功后，智能体必须显示“已记录到人脉地图”或“已更新到人脉地图”的轻量标记。

删除、合并、批量覆盖、共享、导出、对外发送与敏感判断必须获得明确确认。
