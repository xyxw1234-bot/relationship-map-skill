# 人脉地图能力包

当前正式版本：`v3.0.3`。

人脉地图是 Hermes 能力包：插件提供受控关系资产工具，Skill 负责自然语言行为，关系资料不再依赖聊天记忆保存。

## 用户安装

在支持 Hermes 插件的节点引擎中，用户只需要发送本仓库的正式安装链接并说“安装人脉地图”。安装运行时应自动完成：

1. 校验固定版本的能力包；
2. 安装并启用 `relationship-map-vault` 插件；
3. 加载捆绑的 `relationship-map` Skill；
4. 初始化当前用户的独立关系资产空间；
5. 完成一次真实读写自检；
6. 向用户展示简短使用示例。

用户不需要安装数据库、配置路径或选择存储位置。

## 当前包的持久化边界

插件默认使用 Hermes Home 下独立的关系资产库：

`$HERMES_HOME/data/relationship-map/relationship-map.db`

它独立于 Hermes 会话和记忆文件，采用 SQLite 事务、WAL 日志和追加式互动时间线。升级插件不会覆盖这个目录；任何涉及删除、合并、批量覆盖的未来操作必须先创建备份。

Node Engine 托管版可用同一业务工具契约接入 PostgreSQL 用户空间。Skill 与用户使用方式不变；替换的是服务端存储适配器，而不是让终端用户迁移或配置数据库。

## 自动记录标记

普通互动、明确承诺和待跟进事项自动保存后，智能体必须在回复中显示：

- 已记录到人脉地图：……
- 已更新到人脉地图：……

用户可以直接说“这条不该记录”“删除刚才那条”或“修改刚才记录”。删除、合并、共享、导出和对外动作仍需明确确认。
