人脉地图

当前版本：v2.6。

人脉地图是一个面向真实经营场景的人脉管理 Skill。它帮助用户把联系人、沟通记录、会议资料、电话记录、拜访记录、项目资料和合作线索，沉淀成可查、可筛、可判断、可行动的关系资产。

它不只是通讯录。它关注的是：这个人是谁、从哪里认识、和我是什么关系、最近发生过什么、现在适合怎么跟进、能否引荐资源、能否推动项目、是否需要重新维护，以及下一步该采取什么动作。

核心能力包括：

1. 建档与沉淀：把聊天记录、会议纪要、电话沟通、拜访记录、文档资料中的人物、组织、需求、资源、承诺、风险和下一步动作整理进人脉地图。
2. 查看与筛选：按姓名、城市、单位、身份、行业、关系类型、标签、最近互动、联系频次等条件快速找到人。
3. 标签与画像：为联系人沉淀身份角色、关系来源、合作方向、资源类型、关系温度、信任程度、行动优先级等画像信息。
4. 关系判断与排序：判断谁关系更密切、谁正在变生疏、谁适合优先联系、谁适合请教、谁适合合作、谁需要重新触达。
5. 场景行动支持：围绕会前准备、会后沉淀、联系话术、拜访计划、合作推进、转介绍请求和项目跟进，生成可执行建议。
6. 资源路径与机会经营：围绕一个项目、需求、城市、行业或目标，找出可能能引荐、协作、推动、成交或提供信息的人，并区分只是认识、有资源、愿意提供信息、能引荐、能推动和能直接调动。

适用场景：

- 管理客户、合作伙伴、资源方、专家、校友、家长、供应商、渠道和关键影响者。
- 从会议纪要、聊天记录、电话沟通、活动复盘、项目资料中提炼人脉线索。
- 在拜访、开会、转介绍、项目推进、资源整合前，快速了解应该找谁、怎么说、先做哪一步。
- 复盘一段时间的人脉经营情况，发现关系变冷、承诺遗漏、机会未跟进或关键人维护不足的问题。

工作方式：

人脉地图优先使用自然语言交互。用户不需要记字段名、命令或固定格式，可以直接说“找长沙的朋友”“按联系频次排一下”“这个人适合怎么跟进”“根据这个项目帮我找能推动的人”。智能体应先理解语义，再结合已沉淀的联系人资料、互动记录、标签和上下文进行筛选、判断或生成行动建议。

联系人很多时，应先展示一组简要信息，不一次性铺开全部资料。用户可以继续查看后面的人，也可以随时改成按城市、行业、身份、关系温度、最近互动、行动优先级等条件筛选。

边界原则：

- 不凭空编造联系人事实。
- 不把推断写成已确认事实。
- 不默认展示手机号、微信号、详细地址、证件信息、财务备注和私密评价。
- 不在用户未确认前修改、删除、合并或批量导入联系人。
- 对关系可调动度、资源路径和下一步动作的判断，必须说明依据；信息不足时，应提示需要补充什么。

使用方式：把上面的链接发给自己的智能体，或直接说“安装这个能力”。接收方应自动识别为可安装 Skill，完成检查、安装或更新，并在安装后给出能力示例。

安装完成后，可以直接这样问：

- 打开我的人脉地图。
- 帮我找出长沙认识的朋友。
- 哪些人和我关系最密切？
- 哪些人变生疏了，需要重新触达？
- 给这几个人打上“重要合作方”标签。
- 我要和张三开会，帮我做会前准备。
- 这是我和李四的会议纪要，帮我沉淀到人脉地图。
- 根据这个项目目标，帮我找可能能引荐、推动或协作的人。

安全边界：联系人更新、标签修改、合并、删除、批量导入和敏感判断，必须先给出预览并等待用户确认。事实、推断和待确认信息必须区分清楚，不把没有依据的判断写成事实。
