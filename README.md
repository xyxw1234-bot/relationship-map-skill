# 人脉地图

> 面向真实经营场景的可安装关系资产能力包：把联系人、互动、承诺、待办和客户表沉淀为可查询、可排序、可行动的数据，而不是散落在聊天记忆里。

**当前正式版本：`v3.1.0`**

## 它能做什么

### ① 🧭 看清人脉全貌
- 查看当前人脉总数与移动端摘要地图；
- 默认每次展示 20 人，支持“下一页”或明确要求“全部展示”；
- 按最近互动、沟通频次、关系热度、待跟进日期或姓名排序。

### ② 📝 把关系动作沉淀下来
- 会面、通话、拜访、饭局、合作沟通可追加到联系人时间线；
- 明确承诺和待跟进事项自动进入关系库；
- 每次成功保存都会显示“已记录到人脉地图”或“已更新到人脉地图”。

### ③ 🎯 找到该优先经营谁
- 按姓名、单位、城市、岗位、标签和自定义字段检索；
- 基于已保存的互动次数、最近互动时间和待办日期，提供频次、热度与跟进优先级排序；
- 排序只基于已保存数据，不把推断伪装成事实。

### ④ 🤝 让会前会后形成闭环
- 会前读取联系人背景、近期互动、未完成承诺与待办；
- 会后把关键进展、下一步和承诺沉淀回来；
- 跨会话、跨进程仍可读取关系资产。

### ⑤ 📥 管理客户表与分类
- 导入 CSV / XLSX 客户资料，自动识别常见字段；
- 保留原表快照，不默认修改用户源文件；
- 支持标签与自定义字段，适合按项目、区域、客户阶段等维度经营。

## 安装

**固定活链接：**

`https://raw.githubusercontent.com/xyxw1234-bot/relationship-map-skill/main/SKILL.md`

将这个链接发送给支持 Hermes 插件的节点引擎，并说：**“安装人脉地图。”**

- 首次安装会自动完成插件启用、Skill 加载、关系库初始化和自检；
- 已安装用户说“检查更新人脉地图”或“更新人脉地图”，会优先走增量更新；
- 更新只替换插件与 Skill，不覆盖已有联系人、互动、承诺和待办；
- 用户不需要配置 Python、数据库、路径或存储位置。

> Tag 仅用于回滚、审计与复现；正常用户始终使用上面的固定活链接。

## 直接试用

- `打开我的人脉地图。`
- `我现在一共有多少人脉？`
- `按沟通频次帮我排一下。`
- `哪些人最近该重新联系？`
- `后天去长沙见张总，帮我准备一下。`
- `刚和李校长开完会，他对研学合作很感兴趣，帮我记下来。`
- `王总答应下周引荐两位负责人，提醒我跟进。`
- `把这张客户表导入人脉地图。`

## 数据与安全边界

- 每个用户使用独立关系资产空间，默认位于 `$HERMES_HOME/data/relationship-map/`；
- 使用 SQLite 事务、WAL〔预写日志〕与追加式互动时间线；
- 删除、合并、批量覆盖、共享、导出和对外发送必须获得用户明确确认；
- 数据库与插件代码目录分离，正常更新不会覆盖用户关系资产。
