# Changelog

## v3.1.0

- 新增 `relationship_map_overview`：基于持久化关系库返回总人数、移动端联系人摘要、受控分页与下一页游标。
- 新增可解释的关系经营排序：按最近互动、互动频次、关系热度、待跟进到期日或姓名排序；热度只使用已保存的互动次数与最近互动时间计算。
- 默认地图总览每次展示 20 人；只有用户明确要求全部时才连续展示，避免手机端一次展开大量联系人资料。
- 安装入口明确区分首次安装与已安装后的增量检查更新：优先 `plugins update`，失败才做一次代码级强制重装，关系库目录不受更新覆盖。
- 优化大规模地图浏览：互动与待办使用 SQLite 聚合查询，避免按联系人逐条查询；补齐分页、排序、版本契约回归测试。
- 统一固定活链接、安装契约、自动欢迎与运行 Skill 的欢迎说明；安装成功后展示结构化五段式能力总览与可直接发送的使用示例。

## v3.0.4

- 修复用户旅程：打开/查看人脉地图时，`relationship_map_search` 允许空查询并返回当前联系人总览，避免已记录关系被误报为空。
- 增加空查询地图总览回归测试；用户旅程验收覆盖安装、会后记录、地图总览、会前准备与跨进程回读。

## v3.0.3

- 修复发布完整性：统一根安装 Skill、插件清单、运行时 Skill、`pyproject.toml` 与安装契约为 `3.0.3`。
- 清除错误行号前缀、重复且版本漂移的嵌套插件元数据与旧版纯 Skill 副本。
- 固定正式安装入口，补齐插件、命名空间 Skill、九项工具、持久化自检和自动欢迎攻略的验收契约。
- 用户安装入口统一为固定活链接 `main/SKILL.md`；版本 Tag 仅用于回滚、审计与复现。

## v3.0.0

- 人脉地图从纯 Skill 升级为能力包：新增 `relationship-map-vault` Hermes 插件与捆绑 Skill。
- 新增独立关系资产库，不再把联系人、互动、承诺和待办只依赖于聊天记忆。
- 新增联系人查询、详情、互动沉淀、承诺沉淀、待跟进和会前准备六项受控能力。
- 普通互动、明确承诺和待跟进事项自动沉淀，并在用户回复中显示“已记录到人脉地图”或“已更新到人脉地图”。
- 增加用户范围隔离、追加式互动时间线、审计事件与数据库备份能力。
- 用户数据目录与插件代码目录分离；插件升级不覆盖已有关系资产。
- 新增 CSV 与 XLSX 客户资料导入：自动识别常见字段、保存源文件快照、导入批次与自定义字段，支持单个用户 5,000 条客户资料批量导入。
- 新增自由标签与自定义字段分类，可按部门、客户特性、地域、项目阶段等任意用户定义维度查询。
- 自动导入不会修改用户原始表格；只有用户明确要求同步时才进入源表更新流程。
